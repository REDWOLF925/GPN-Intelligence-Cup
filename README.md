# GPN-Intelligence-Cup
Data scientist в управлении проектной командой

## Структура репозитория:
```bash
.
├── data/ # Директория для хранения датасетов на всех этапах их обработки.
│   ├── cases.csv # Исходный датасет с описанием заказов.
│   ├── cases_labeled.csv # Размеченный исходный датасет с описанием заказов.
│   ├── diaries.csv # Исходный датасет с дневниками героев.
│   ├── marks.csv # Исходный датасет с оценками героев.
│   ├── stats.csv # Датасет, содержащий статистические данные по героям (средние оценки, среднее время на выполнение дейсвтий и т.д.).
│   └── test.csv # Датасет сгенерированных описаний заказов с метками (для валидации модели).
│
├── models/ # Директория для хранения сериализованных с помощью пакета joblib моделей.
│   └── classification_model.pkl # pipeline (TF-IDF -> LogisticRegression) для классификации заказов.
│
├── notebooks/ # Директория для хранения упорядоченного набора ноутбуков.
│   ├── 0_data_labeling.ipynb # Автоматическая (почти) разметка данных.
│   ├── 1_classification_cases.ipynb # Классификация типов поручений.
│   ├── 2_data_analysis.ipynb # Анализ данных, формирование датасета статистики по Героям.
│   └── 3_create_team.ipynb # Формирование команд на ещё не выполненных заказах.
│
├── history.docx # Описание кейса.
├── history.pdf # Описание кейса.
└── README.md
```
